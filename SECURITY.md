# Security Policy

## Supported Versions

Security fixes are applied to the latest supported release line and the
current `main` branch.

| Version | Supported |
| --- | --- |
| Latest published `0.x` release | Yes |
| `main` branch | Yes |
| Older releases | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities in public GitHub issues or
public discussions.

Use GitHub's private vulnerability reporting for this repository when it is
available. If private reporting is not enabled, contact the maintainers
through a private channel associated with the repository owner instead of
opening a public issue.

When reporting an issue, include:

- A clear description of the affected package or workflow
- Reproduction steps or a proof of concept when possible
- Impact details and any suggested mitigations

We aim to acknowledge new reports within 5 business days and provide an
initial assessment within 10 business days. Timelines for remediation and
disclosure will vary based on severity and release readiness.
